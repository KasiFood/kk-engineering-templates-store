KK ENGINEERING SECURE SALES FLOW — SYSTEM B
Secure token-based download after PayFast ITN verification

WHAT CHANGED
1. The website no longer gives a direct /downloads/ link after PayFast returns the customer.
2. PayFast checkout links are now generated by a Netlify Function: /api/create-checkout
3. PayFast ITN posts to: /api/payfast-itn
4. A verified order is stored in Netlify Blobs.
5. The download page polls /api/get-download-link?order=...
6. The actual file is served only by /api/download-file?token=...
7. Direct browser access to /downloads/* is blocked by netlify.toml.

NETLIFY ENVIRONMENT VARIABLES
Set these in Netlify Dashboard → Site configuration → Environment variables:

PAYFAST_MERCHANT_ID       = your PayFast merchant ID
PAYFAST_MERCHANT_KEY      = your PayFast merchant key
PAYFAST_PASSPHRASE        = your PayFast passphrase, if configured in PayFast; otherwise leave blank
PAYFAST_LIVE              = false for sandbox testing, true for live payments
LINK_SECRET               = any private random 32+ character string
FROM_EMAIL                = khoaranekatleho@gmail.com, or your verified sending email
SENDGRID_API_KEY          = optional; only needed if you want automatic download emails

Generate LINK_SECRET with one of these:
- openssl rand -hex 32
- or use any password generator and create a 32+ character random string

PAYFAST MODE
Sandbox test:
PAYFAST_LIVE=false
Use sandbox Merchant ID and Merchant Key.

Live test:
PAYFAST_LIVE=true
Use live Merchant ID and Merchant Key.

IMPORTANT
Do not test this by opening index.html directly from your computer. The secure flow needs Netlify Functions.
Deploy to Netlify or run locally with Netlify CLI using:

npm install
netlify dev

TEST FLOW
1. Open the deployed site.
2. Click Buy on one product.
3. payment.html opens.
4. Enter an email address.
5. Click Pay with PayFast.
6. /api/create-checkout creates the PayFast URL.
7. Complete payment in PayFast.
8. PayFast sends ITN to /api/payfast-itn.
9. The function verifies signature, merchant ID, amount, status, and PayFast server validation.
10. The function stores the verified order and signed token.
11. PayFast returns the customer to download.html?order=...
12. download.html polls /api/get-download-link until verification is complete.
13. The user clicks the secure download button.
14. /api/download-file validates the token and serves the file.

DIRECT DOWNLOAD BLOCKING
The following should return 404:
/downloads/maintenance-planning-excel.xlsx

The following should only work with a valid token:
/api/download-file?token=...

NOTES
- SendGrid is optional. Without SENDGRID_API_KEY, the download still works on the returned download page.
- Download tokens expire after 48 hours.
- For high sales volume, consider moving orders to Supabase or a full ecommerce system later. For your current template store, Netlify Blobs is a suitable lightweight order store.
